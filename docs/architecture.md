# Obsidian MCP Hybrid Ecosystem Plan

## Goal
Establish a high-performance, modular MCP ecosystem that separates visual knowledge editing (Excalidraw) from vault-wide semantic intelligence (Companion).

## Architecture Overview

### 1. Obsidian Excalidraw MCP (Standalone CLI)
- **Status**: Current project (`obsidian-excalidraw-mcp`).
- **Nature**: Standalone Node.js CLI.
- **Responsibility**: 
  - Deep editing of `.excalidraw.md` files.
  - No dependency on Obsidian UI/State.
  - Specialized for power users/Excalidraw users.
- **Why Standalone?**: Modularity. Users who don't use Excalidraw don't need this. File-based operations are sufficient.

### 2. Obsidian Companion MCP (Hybrid: Plugin + MCP)
- **Status**: New project (to be created).
- **Nature**: Obsidian Plugin + Thin Node.js MCP.
- **Components**:
  - **The Indexer (Plugin)**: Background indexing (mtime check), Semantic Embedding generation (Transformers.js or API), Vector DB management (lancedb-js or similar).
  - **The MCP (NPM/npx)**: Lightweight CLI that talks to the plugin via localhost (WebSocket/HTTP). Standardizes `stdio` for the Agent.
- **Responsibility**:
  - Vault-wide Semantic Search.
  - Note CRUD & Frontmatter management.
  - Contextual awareness (knowing what's "open" in Obsidian).
- **Why Hybrid?**: Solves the "heavy indexing" problem during the short lifecycle of an MCP call. Offloads processing to the persistent Obsidian app.
- **Context Awareness (Confirmed)**: The Obsidian API provides full access to the active editor (`MarkdownView`), cursor position (`getCursor`), and unsaved content (`getValue`). This allows the AI to perform precise edits like "insert after the current paragraph."

## Technical Feasibility: Editor Context Access

Research confirms that an Obsidian Plugin can expose the following capabilities to the MCP MCP:
| Capability | Obsidian API Method | AI Use Case |
| :--- | :--- | :--- |
| **Get Active Note** | `app.workspace.getActiveViewOfType(MarkdownView)` | Know which file the user is looking at. |
| **Get Cursor Position** | `editor.getCursor()` | Identify where the user is typing right now. |
| **Get Unsaved Content** | `editor.getValue()` | Access changes before they are saved to disk. |
| **Insert Text** | `editor.replaceRange(text, pos)` | Insert links/content precisely at the cursor or specific line. |
| **Get Selection** | `editor.getSelection()` | Rephrase or summarize the text the user has highlighted. |

### Data Contract (MCP <-> Plugin)
The MCP will forward JSON-RPC requests to the Plugin. 
- **Protocol**: WebSocket or HTTP (REST).
- **Security**: Localhost-only + API Key generated by the Plugin.

### Language & Stack
- **Languages**: 100% TypeScript (ESM).
- **Semantic Engine**: `Transformers.js` (for local embeddings in Obsidian) or OpenAI API.
- **Vector DB**: `lancedb` (if file storage) or in-memory vector indexing for smaller vaults.

## User Workflow
1. User installs **Obsidian Companion Plugin**.
2. User adds **Companion MCP** to AI Agent settings.
3. User adds **Excalidraw MCP** (current project) to AI Agent settings.
4. AI Agent orchestrates between the two:
   - "Find related notes via Companion."
   - "Update the diagram via Excalidraw MCP."
